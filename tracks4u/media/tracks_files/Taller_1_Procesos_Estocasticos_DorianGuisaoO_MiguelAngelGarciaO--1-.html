Problema 1: Simulación MCMC y verificación de la distribución estacionaria

Tenemos la siguiente cadena de Markov

\[x_t = \beta x_{t-1} + \varepsilon_t, \quad \varepsilon_t \overset{iid}{\sim} N(0,1), \quad \beta = 0.1\]

y la siguiente distribución estacionaria teórica:

\[X_t \sim N\left(0,\, \frac{1}{1-\beta^2}\right) = N(0,\, 1.0101)\]

El objetivo es verificar empíricamente, mediante métodos gráficos y tests formales, que la simulación efectivamente converge a esta distribución.

Parámetros y simulación

beta    <- 0.1
n       <- 20000
burn_in <- 2000
x <- numeric(n)
epsilon <- rnorm(n, mean = 0, sd = 1)
for (t in 2:n) {
  x[t] <- beta * x[t-1] + epsilon[t]
}

# Eliminamos el período de burn-in
x_stationary <- x[(burn_in + 1):n]

Los valores teóricos de media y varianza de la distribución estacionaria son:

mean_teorica <- 0
var_teorica  <- 1 / (1 - beta^2)
sd_teorica   <- sqrt(var_teorica)

cat("Media teorica:   ", mean_teorica)
## Media teorica:    0
cat("\nVarianza teorica:", var_teorica)
## 
## Varianza teorica: 1.010101

1. Convergencia de la cadena

plot(x, type = "l", col = "steelblue",
     main = "Trace Plot MCMC",
     xlab = "Iteracion", ylab = "Valor de X_t")
abline(v = burn_in,      col = "red",       lwd = 2, lty = 2)
abline(h = mean_teorica, col = "darkgreen", lwd = 2)
legend("topright", legend = c("Burn-in", "Media teorica"),
       col = c("red", "darkgreen"), lty = c(2, 1), lwd = 2)

El trace plot muestra que la cadena mezcla rápidamente y oscila de forma estable alrededor de la media teórica \(\mu = 0\) desde las primeras iteraciones. Esto es consecuencia directa de que \(\beta = 0.1\) es pequeño, lo que implica una autocorrelación muy débil entre iteraciones consecutivas. En otras palabras, la cadena olvida casi de inmediato su estado anterior. Este comportamiento es la primera señal de que la cadena ha alcanzado su distribución estacionaria.

Para confirmar esta hipótesis, recurrimos a métodos gráficos y tests formales en las secciones siguientes.

El histograma de los valores post burn-in se ajusta visualmente de forma muy precisa a la curva de densidad \(N(0,\, 1.0101)\) (línea roja). La distribución empírica es simétrica, unimodal y centrada en cero, sin indicios de asimetría ni colas pesadas. Esto constituye una primera confirmación gráfica de que la distribución estacionaria es la normal teórica.

El QQ-plot compara los cuantiles empíricos de la muestra contra los cuantiles teóricos de una distribución normal. Las leves desviaciones en los extremos (\(\pm 4\)) son esperables en muestras grandes y no comprometen la conclusión. Este gráfico refuerza que la distribución estacionaria es normal.

El estadístico \(W = 0.99974\) es extremadamente cercano a 1, y el p-value \(= 0.81\) es muy superior al nivel de significancia usual de \(0.05\). Por lo tanto, no se rechaza \(H_0\) de normalidad. No hay evidencia estadística para descartar que los datos provengan de una distribución normal.

Esto confirma que la distribución empírica es consistente con la \(N(0,\, 1.0101)\) teórica.

Conclusión

El conjunto de evidencias presentadas. Convergencia del trace plot, concordancia de momentos, ajuste del histograma, linealidad del QQ-plot y resultados de los tests formales apunta de forma consistente a la conclusión esperada: la distribución estacionaria de la cadena \(x_t = 0.1\, x_{t-1} + \varepsilon_t\) es efectivamente

\[N\!\left(0,\, \frac{1}{1-\beta^2}\right) = N(0,\, 1.0101)\]

Problema 2: Simulación de una Beta(2,6) via Metropolis-Hastings

En este problema la densidad objetivo es una \(\text{Beta}(2, 6)\):

\[f(x) = \frac{x^{\alpha-1}(1-x)^{\beta-1}}{B(\alpha,\beta)}, \quad x \in (0,1), \quad \alpha=2,\ \beta=6\]

con media teórica \(\mu = \frac{\alpha}{\alpha+\beta} = 0.25\) y varianza \(\sigma^2 = \frac{\alpha\beta}{(\alpha+\beta)^2(\alpha+\beta+1)} \approx 0.026\).

Implementación

set.seed(123)
n          <- 30000
burn_in    <- 3000
sigma_prop <- 0.22

f <- function(x) dbeta(x, 2, 6)

Se utiliza una caminata aleatoria normal como distribución propuesta: \(Y_t \sim N(x_t,\ \sigma^2_{prop})\) con \(\sigma_{prop} = 0.22\). La elección de esta propuesta responde a dos criterios. Primero, el algoritmo de Metropolis-Hastings requiere asociar a la densidad objetivo \(f\) una densidad propuesta \(q(\cdot | y)\) que sea fácil de simular y cuyo soporte contenga al soporte de \(f\); la normal cumple ambas condiciones, ya que es trivial simular de ella y su soporte es \(\mathbb{R}\), el cual contiene al soporte \((0,1)\) de la \(\text{Beta}(2,6)\). Segundo, la caminata aleatoria normal es una de las propuestas más utilizadas en implementaciones prácticas de MH, lo que la convierte en una elección sencilla como punto de partida.

# Inicialización
x <- numeric(n)
x[1] <- 0.5   # Punto de inicio
accept <- 0


# Algoritmo Metropolis-Hastings
for (t in 1:(n-1)) {
  
  # Caminata aleatoria normal
  y <- rnorm(1, mean = x[t], sd = sigma_prop)
  
  # Verificación de soporte y razón de aceptación
  prob_y <- f(y)
  prob_x <- f(x[t])
  
  if (prob_y > 0) {
    rho <- min(prob_y / prob_x, 1)
    
    if (runif(1) < rho) {
      x[t+1] <- y
      accept <- accept + 1
    } else {
      x[t+1] <- x[t]
    }
  } else {
    # Fuera de soporte: Rechazo automático
    x[t+1] <- x[t]
  }
}

# Tasa de aceptación

tasa_aceptacion <- accept / n
cat("Tasa de aceptacion final:", round(tasa_aceptacion, 3), "\n")
## Tasa de aceptacion final: 0.575

La tasa de aceptación obtenida es de 57.5%. Esto indica que \(\sigma_{prop}\) podría aumentarse ligeramente para proponer saltos más ambiciosos, aunque como veremos, los métodos gráficos y test confirman que los resultados son válidos.

Post-procesamiento: Burn-in y Thinning

x_after_burn <- x[(burn_in + 1):n]
x_mcmc       <- x_after_burn[seq(1, length(x_after_burn), by = 20)]

Se eliminan las primeras 3000 iteraciones como período de burn-in. Adicionalmente, se aplica thinning, reteniendo 1 de cada 20 muestras. Esta decisión responde a dos motivos relacionados.

El primero es la autocorrelación: en MH con caminata aleatoria, cuando el algoritmo rechaza un salto propuesto, el valor anterior se repite exactamente. Esto genera dependencia entre iteraciones consecutivas que puede sesgar los diagnósticos. El segundo motivo es más técnico: esa repetición exacta de valores produce empates (Se producen aquí: # Fuera de soporte: Rechazo automático x[t+1] <- x[t]) en la muestra, lo cual es problemático para el test de Kolmogorov-Smirnov, que asume variables continuas donde la probabilidad de observar dos valores idénticos es cero. La presencia de empates genera escalones artificiales en la función de distribución empírica e invalida el cálculo estándar del estadístico \(D\). El thinning resuelve ambos problemas simultáneamente.

1. Convergencia

plot(x_mcmc, type = "l", col = "steelblue",
     main = "Trace Plot (Thinned)", xlab = "Muestra", ylab = "Valor")
abline(h = 2/(2+6), col = "darkgreen", lty = 1)
legend("topright", legend = "Media teorica", col = "darkgreen", lty = 1)

La cadena mezcla bien y oscila establemente alrededor de la media teórica \(\mu = 0.25\) (línea verde) a lo largo de todas las muestras post burn-in. No se observan períodos de estancamiento ni tendencias sistemáticas, lo que es un primer indicio de que la cadena ha alcanzado su distribución estacionaria.

2. Autocorrelación

par(mfrow = c(1, 2))

acf(x_after_burn, main = "ACF sin Thinning", lag.max = 40)
acf(x_mcmc,       main = "ACF tras Thinning", lag.max = 40)

par(mfrow = c(1, 1))

La comparación entre ambos gráficos es ilustrativa. Sin thinning, se observan autocorrelaciones significativas en los primeros lags — donde el lag \(k\) mide la correlación entre una observación y la que ocurrió \(k\) pasos atrás — que decaen gradualmente: las muestras consecutivas están correlacionadas precisamente porque el algoritmo tiende a quedarse en el mismo valor cuando rechaza propuestas. Tras aplicar thinning de 1 cada 20, todas las barras caen dentro de las bandas de confianza desde el lag 1, indicando que la autocorrelación es estadísticamente indistinguible de cero. Las muestras retenidas pueden tratarse como independientes, lo cual valida el uso del test KS como se vera después.

3. Histograma vs densidad teórica

     main = "Histograma vs Teorica", xlab = "x")
curve(dbeta(x, 2, 6), col = "darkred", lwd = 2, add = TRUE)

El histograma reproduce la forma característica de la \(\text{Beta}(2,6)\): asimétrica a la derecha, con moda cerca de \(x = 0.2\) y cola larga hacia valores altos. El ajuste a la curva teórica es visualmente bueno, lo que constituye un indicio de que la simulación está capturando correctamente la distribución objetivo.

A diferencia del Problema 1, aquí el QQ-plot no compara contra una normal sino directamente contra cuantiles simulados de una \(\text{Beta}(2,6)\), que es la comparación correcta para este caso en específico. Los puntos siguen la línea de referencia con buena precisión en la mayor parte del rango. Las pequeñas desviaciones en la cola superior (\(x > 0.6\)) son esperables dada la escasez de observaciones en esa región, y no representan una desviación sistemática preocupante, aunque si es indica que la exploración en esa zona es baja o que el algoritmo subestima la probabilidad en esa zona.

5. Test de Kolmogorov-Smirnov

print(ks_test)
## 
##  Asymptotic one-sample Kolmogorov-Smirnov test
## 
## data:  x_mcmc
## D = 0.021799, p-value = 0.5426
## alternative hypothesis: two-sided

El test reporta \(D = 0.0218\) y \(\text{p-value} = 0.5426\). La distancia máxima entre la distribución empírica y la \(\text{Beta}(2,6)\) teórica es muy pequeña, y el p-value es ampliamente superior al nivel de significancia usual de \(0.05\). No se rechaza \(H_0\), lo que indica que los datos simulados son estadísticamente consistentes con la distribución objetivo.

Conclusión

El conjunto de diagnósticos aplicados apunta de forma consistente a la misma conclusión: el algoritmo de Metropolis-Hastings implementado simula correctamente variables aleatorias de una \(\text{Beta}(2, 6)\). El trace plot confirma estacionariedad, la comparación de ACF confirma que el thinning eliminó la autocorrelación, el histograma y QQ-plot muestran buen ajuste a la forma teórica, y el test KS formaliza que no existe diferencia estadísticamente significativa entre la muestra MCMC y la distribución objetivo.

Problema 3: Metropolis Hastling

Muestra aleatoria

Se genera una muestra aleatoria de tamaño 100 proveniente de la distribución mezclada

\[ f(y \mid \alpha) = (1-\alpha)\, f_{\mathcal{N}(8,5)}(y) + \alpha\, f_{t_{3}(10)}(y) \]

La simulación se realiza generando primero una muestra Bernoulli con probabilidad de éxito \(\alpha = 0.7\).
Para cada observación:

  • Si el resultado es 1, se genera un valor de una distribución \(t\) con \(\nu = 3\) centrada en 10.
  • Si el resultado es 0, se genera un valor de una distribución Normal con media 8 y desviación estándar 5.

Calculo de funcion de verosimilitud

Calculamos \(p(y|\alpha)\) usando los datos ya simulados

verosimilitud <- function(alpha, y){
  
  densidades <- (1 - alpha) * dnorm(y, mean = 8, sd = 5) +
                 alpha * dt(y - 10, df = 3)
  
  prod(densidades)
}

Algoritmo Metropolis–Hastings

Tenemos la verosimilitud \(p(y \mid \alpha)\), pero nuestro interés es estimar la distribución posterior \(p(\alpha \mid y)\), ya que esta describe la proporción de observaciones provenientes de la distribución t-Student (\(\nu=3\)) frente a la Normal\((8,5)\).

Aplicando el Teorema de Bayes:

\[ p(\alpha \mid y) = \frac{p(\alpha)\,p(y \mid \alpha)}{p(y)}, \]

y como \(p(y)\) no depende de \(\alpha\), obtenemos

\[ p(\alpha \mid y) \propto p(\alpha)\,p(y \mid \alpha) = p(\alpha)\,L(\alpha), \]

donde \(p(\alpha)\) es la distribución a priori y \(L(\alpha)\) es la verosimilitud.

Dado que esta distribución posterior no tiene forma cerrada, utilizamos el algoritmo Metropolis–Hastings, el cual construye una cadena de Markov cuya distribución estacionaria es \(p(\alpha \mid y)\).

En cada iteración se propone un nuevo valor \(\alpha^*\) y se acepta con probabilidad

\[ \rho = \min\left( \frac{p(\alpha^*) L(\alpha^*)} {p(\alpha^{(t)}) L(\alpha^{(t)})}, 1 \right). \]

De esta manera se generan muestras aproximadas de la distribución posterior, permitiendo estimar cantidades como

\[ E(\alpha \mid y). \]

  • Creamos la funcion prior y posterior
prior <- function(alpha, a, b){
  dbeta(alpha, a, b)
}

posterior <- function(alpha, y, a, b){
  
  if(alpha <= 0 || alpha >= 1) return(0) # debido a que 0<=alpha<=1
  
  prior(alpha, a, b) * verosimilitud(alpha, y)
}
  • Inicializamos la MCMC con 10.000 iteraciones, un \(\alpha_0\) = 0.5 y \(\sigma\) = 0.05
n_iter <- 10000
alpha_chain <- numeric(n_iter)
alpha_chain[1] <- 0.5
sigma_prop <- 0.05
y <- MA

  • Implementación de Metropolis–Hastings para beta(1,1)
for(t in 2:n_iter){
  
  alpha_current <- alpha_chain[t-1]
  alpha_proposed <- rnorm(1, alpha_current, sigma_prop)
  
  p <- posterior(alpha_proposed, y, a1, b1) /
       posterior(alpha_current, y, a1, b1)
  
  p <- min(p, 1)
  
  if(runif(1) < p){
    alpha_chain[t] <- alpha_proposed
  } else {
    alpha_chain[t] <- alpha_current
  }
}
